Overview

AI agents are stateless by default. Every session begins from scratch — no memory of what was decided last week, no awareness of what another agent resolved yesterday, no trail of evidence for the decisions that matter most. At scale, multi-agent systems become incoherent: different agents hold contradictory beliefs about the same entity, and no one can explain why.

RECALL is a shared memory operating system for AI agents. It provides a persistent, governed memory layer that agents read from and write to across sessions, deployments, and model versions. Every write is validated against a workspace policy, every decision produces a tamper-evident receipt, and every contradiction is surfaced as a conflict before it causes harm.

RECALL is built on Walrus for permanent decentralized blob storage, Sui for on-chain governance and receipt anchoring, and Seal for threshold encryption that ties memory access to on-chain identity. The control plane runs as a gRPC + REST service that integrates with any agent framework via a Python or TypeScript SDK.

Quickstart

  1. Start the control plane
    Runs gRPC on :9090 and REST on :8080
    cargo run -p recall-control-plane
  2. Seed demo data
    Creates workspaces, agents, memory entries, and conflicts
    python demo_seed.py
  3. Stream live writes
    Follow memory events in real time with conflict markers
    recall logs --follow
  4. Check unresolved conflicts
    List all contradictions that need human or automated resolution
    recall failures
  5. Audit an entity's memory trail
    Full receipt chain with conflict annotations
    recall why --entity sarah@email.com

SDK Reference

Install the Python SDK: pip install recall-sdk

recall.connect() CONNECTION
Initialize a connection to the RECALL control plane.
import recall

client = recall.connect(
    endpoint="localhost:9090",
    workspace="ws_customer-ops",
    passport_id="pp_fraud_agent_001",
)
client.workspace.read(entity) READ
Read all memory entries for an entity in the current workspace.
entries = client.workspace.read("sarah@email.com")

for e in entries:
    print(e.event, e.data, e.agent_id, e.timestamp)
client.workspace.write(entity, event, data) WRITE
Write a memory entry. Triggers governance check, conflict detection, and receipt emission.
receipt = client.workspace.write(
    entity="sarah@email.com",
    event="fraud_assessment",
    data={
        "flagged": True,
        "score": 0.91,
        "reason": "velocity_anomaly",
    },
    tags=["pii", "fraud"],
    trust_level=2,
)

print(receipt.id)          # cryptographic receipt ID
print(receipt.walrus_blob) # permanent Walrus blob ID
client.recall.handoff(target_agent, entity) HANDOFF
Package all memory for an entity into a handoff capsule and deliver it to another agent. The capsule is encrypted with Seal and stored on Walrus.
capsule = client.recall.handoff(
    target_agent="escalation-agent",
    entity="sarah@email.com",
    include_conflicts=True,
)

print(capsule.blob_id)  # Walrus blob ID for the capsule
print(capsule.receipt)  # anchored handoff receipt
client.recall.publish(name, version) REGISTRY
Publish the current workspace's memory as a versioned package in the RECALL registry.
pkg = client.recall.publish(
    name="fraud-detection-agent",
    version="1.0",
    category="fraud",
    description="Fraud detection memory from 6 months production",
)

print(pkg.name, pkg.memory_count)
# fraud-detection-agent@1.0  847 memories

CLI Reference

COMMAND FLAGS DESCRIPTION
recall logs --follow, --workspace Stream memory write events. Conflicts are marked inline with a ← CONFLICT marker.
recall failures --workspace List all unresolved conflict records across the workspace.
recall why --entity, --workspace Show the full receipt chain for an entity with inline conflict and escalation annotations.
recall inspect <receipt-id> Display full detail for a single receipt: agent, timestamp, Walrus blob, Sui tx, Seal status.
recall agents --workspace List all agents in the workspace with their role, trust level, enforcement stage, and reputation score.
recall rollback --workspace, --to Roll back all memory entries written after the given Unix timestamp. Emits a rollback receipt.
recall export --entity, --format Export a PDF or JSON audit trail for an entity. Includes memory entries, receipts, Merkle proofs.
recall workspace list — List all workspaces with memory count, agent count, conflict count, and receipt count.
recall workspace create --name, --topology Create a new named workspace with the given topology mode.
recall workspace add-agent --workspace, --agent, --role Add an agent to a workspace with the specified role (READER, WRITER, SUPERVISOR, ADMIN).
recall registry list --category Browse published memory packages in the RECALL registry.
recall registry inspect <name@version> Show package metadata: author, description, memory count, Merkle root, import count.
recall registry import <name@version>, --workspace Import a registry package into the current workspace. Issues a passport for the agent.
recall registry publish --name, --version, --category Publish the current workspace's memory as a versioned registry package.

Architecture

  ┌──────────────────────────────────────────────┐
  │              SDK LAYER                       │
  │  Python SDK · TypeScript SDK · gRPC clients  │
  └──────────────────┬───────────────────────────┘
                     │
  ┌──────────────────▼───────────────────────────┐
  │          RECALL CONTROL PLANE                │
  │          gRPC :9090  REST :8080              │
  │                                              │
  │  AdmissionService    CapabilityService       │
  │  MemoryService       WorkspaceService        │
  │  HandoffService      InspectorService        │
  │  RegistryService     EnvelopeService         │
  └────┬──────────┬──────────────┬───────────────┘
       │          │              │
  ┌────▼───┐ ┌────▼────┐ ┌──────▼──────┐
  │WALRUS  │ │  SUI    │ │   SEAL      │
  │        │ │         │ │             │
  │MemWal  │ │receipt  │ │ threshold   │
  │blob    │ │anchor   │ │ encryption  │
  │storage │ │ Move    │ │ key server  │
  └────────┘ └─────────┘ └─────────────┘
  • SDK Layer Python and TypeScript clients that wrap the gRPC and REST interfaces. Agents use these to read/write memory, request handoffs, and query receipts.
  • Control Plane The core RECALL service. Validates writes against governance rules (evaluated against the Sui WorkspacePolicy object), detects conflicts between concurrent writes, emits cryptographically signed receipts, and coordinates Walrus storage and Sui anchoring.
  • Walrus (MemWal) All memory entries are serialized as protobuf and stored as blobs on Walrus testnet. The blob ID is attached to every memory entry and receipt, providing permanent, verifiable storage independent of the control plane.
  • Sui (Receipt Anchoring) Receipt batches are Merkle-rooted and committed on-chain via the receipt_anchor::commit_anchor() Move function. The on-chain event is the ground truth for audit purposes.
  • Seal (Threshold Encryption) Memory blobs are encrypted using workspace-scoped keys managed by the Seal key server. Decryption requires the agent to hold a valid passport that has been admitted to the workspace policy on Sui.

Move Contracts

workspace_governance

Owns all write-access rules. Called via sui_devInspectTransactionBlock before any memory write is processed.

STRUCT WorkspacePolicy — topology mode, min trust level, constitution version, agent capabilities map
STRUCT AgentEnforcementRecord — deny count, enforcement stage (NONE → DETECT → COACH → QUARANTINE → EVICT), reputation score
FUN check_write_access(policy, record, trust_level, role, tags, scope, has_supervisor) → bool
receipt_anchor

Commits Merkle roots of receipt batches on-chain. Each commit creates an immutable audit event.

STRUCT AnchorRegistry — shared object, stores committed anchor records indexed by merkle root
FUN commit_anchor(merkle_root, walrus_blob_id, workspace_id, receipt_count, registry, clock)
EVENT AnchorCommitted — { merkle_root, walrus_blob_id, workspace_id, receipt_count, committed_at }
workspace_registry

Manages workspace ownership and capability tokens. Grants and revokes agent access via transferable capability objects.

STRUCT WorkspaceOwnerCap — transferable capability, owner has full workspace admin rights
STRUCT AgentCapability — role-scoped, non-transferable capability issued to agent passport
FUN create_workspace(name, topology, clock) → (Workspace, WorkspaceOwnerCap)
FUN grant_agent_capability(workspace, passport_id, role, owner_cap)
