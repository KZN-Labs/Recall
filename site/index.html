THE PROBLEM
01 — THEY FORGET
Every session starts blank. An agent that spent six months learning your customers, your edge cases, your patterns — gone the moment the context window closes. The next session starts from zero. You're not running an agent. You're running a very expensive stateless function.
02 — THEY CONTRADICT
Support approves the credit. Fraud flags the same customer two minutes later. Billing processes it anyway. None of them knew what the others did. Nobody wrote it down. The customer calls back tomorrow with the same problem. This is the default behavior of every multi-agent system in production today.
03 — NOTHING IS AUDITABLE
When an agent makes a wrong decision, the answer to "why did it do that?" is almost always: we don't know. The memory lived in a context window, got flushed, and is gone. In regulated industries this is becoming a compliance crisis. Regulators are starting to ask for decision trails. Current architectures cannot provide them.

USE CASES
CUSTOMER OPS
Support, billing, and fraud agents sharing one workspace. Support resolutions inform billing decisions. Fraud flags prevent credits from being exploited. Compliance exports in minutes, not weeks.
DEFI COORDINATION
Research, risk, and execution agents where every order is traceable to the exact memory that informed it. Every market observation permanently receipted. Full audit trail without extra instrumentation.
HEALTHCARE TRIAGE
Diagnostic agents where contradictory assessments are flagged for human review before any recommendation is made. The full decision trail available for medical audit.
REGULATED WORK
Finance, insurance, legal — any domain where you need to prove afterward exactly what happened, which agent decided it, and what memory it was operating on. Governance rules enforced by Move contracts on Sui.
